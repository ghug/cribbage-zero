Browser tools

Training monitor →worker.html — read-only: live net progress (games/iter), data-bus queue depth, and learner-lease status. Self-play runs in the native cz_pc engine, not the browser. On-device actor →actor.html — in the Android app only: self-play in the background on this phone and upload shards to the bus, with a live readout. A browser can't self-play (shows install info). Evaluate the net →eval.html — 10000 balanced games vs random + vs hard; recording to the progress branch is a separate manual approval. View progress →progress.html — graphs net win% over games trained from the progress branch. Net snapshots & rollback →snapshot.html — archive the net to the net-archive branch, or roll back to a saved snapshot. Refresh the game's net →refresh-trainer.html — triggers cribbage-trainer's workflow to ship the latest net to its "Zero" bot. Release the game →release-trainer.html — launches a full cribbage-trainer release (signed APK). Confirms first.

Command-line tools

Evaluate & record strength

Plays 10000 balanced games vs random + 10000 vs the hard bot, then asks before appending a point to progress-random.csv / progress-hard.csv on the progress branch (y/N). Recording needs ≥ 10000 games.

export CZ_TOKEN=****                    # GitHub PAT (Contents: read+write)
node engine/eval_zero.js [decks=5000]   # prompts before recording
node engine/eval_zero.js --dry          # eval & print, never record (no token needed)
node engine/eval_zero.js --yes          # approve non-interactively
Snapshot / roll back the net

Archive the current net to net-archive (a restore point), list snapshots, or roll one back onto the live net branch. The net branch itself keeps no history, so these are your backups.

export CZ_TOKEN=****                         # GitHub PAT
node engine/snapshot_net.js                  # archive now
node engine/snapshot_net.js --list           # list snapshots (no token needed)
node engine/snapshot_net.js --rollback NAME  # restore (asks y/N)
Contribute self-play from a computer (learner / actor) — native C++ (cz_pc)

Multi-core headless self-play, in C++ (the JS engine/az_contribute.js path is retired). As the learner it trains a bounded replay buffer with shuffled mini-batches, drains the data bus, and pushes the net every ~10k games (+ wind-down). With --actor it only self-plays and uploads sample shards to the bus. One learner, many actors — see docs/BUS.md.

export CZ_TOKEN=****                                         # GitHub PAT (Contents: read+write)
export CZ_BUS_URL=https://cribbage-zero-bus.gabrielhug.workers.dev
export CZ_BUS_TOKEN=****                                     # trainer token (learner) / worker token (actor)

cmake -B core/build -S core && cmake --build core/build -j   # build cz_pc (needs libcurl)
core/build/cz_pc [gamesPerRound=500] [sims=40]   # learner — uses the bus if CZ_BUS_* are set (unset them for solo)
core/build/cz_pc --actor                         # actor (needs CZ_BUS_*; CZ_TOKEN optional)
core/build/cz_pc --dry                           # local bench, no network (--fresh to start over)
# more env: CZ_WORKERS CZ_LR CZ_SIMS CZ_PUSH_GAMES CZ_BUF CZ_BATCH CZ_SHARD_MAX CZ_BUS_LIMIT CZ_MOMENTUM CZ_WD CZ_BRANCH

eval_zero.js's hard bot is a vendored snapshot of the cribbage-trainer heuristic (src/winprob.js + src/engine.js's pegChooseDeep). Version · public domain (The Unlicense).
