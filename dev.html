♣ Cribbage Zero — Tools

A hub for the trainer pages and the command-line dev tools. The trained net lives on the net branch; strength curves on the progress branch.

Browser tools

On-device trainer →local.html — self-plays + trains on this device, pushes the net to GitHub. Cloudflare worker mode →worker.html — contributes self-play to the (optional) Cloudflare data bus. Evaluate the net →eval.html — 1000 balanced games vs random + vs hard; recording to the progress branch is a separate manual approval. View progress →progress.html — graphs net win% over games trained from the progress branch. Net snapshots & rollback →snapshot.html — archive the net to the net-archive branch, or roll back to a saved snapshot. Refresh the game's net →refresh-trainer.html — triggers cribbage-trainer's workflow to ship the latest net to its "Zero" bot. Release the game →release-trainer.html — launches a full cribbage-trainer release (signed APK). Confirms first.

Command-line tools

Evaluate & record strength

Plays 1000 balanced games vs random + 1000 vs the hard bot, then asks before appending a point to progress-random.csv / progress-hard.csv on the progress branch (y/N). Recording needs ≥ 1000 games.

CZ_TOKEN=<github-pat> node engine/eval_zero.js [decks=500]   # prompts before recording
node engine/eval_zero.js --dry          # eval & print, never record
CZ_TOKEN=<pat> node engine/eval_zero.js --yes      # approve non-interactively
Snapshot / roll back the net

Archive the current net to net-archive (a restore point), list snapshots, or roll one back onto the live net branch. The net branch itself keeps no history, so these are your backups.

CZ_TOKEN=<pat> node engine/snapshot_net.js                    # archive now
node engine/snapshot_net.js --list                             # list snapshots
CZ_TOKEN=<pat> node engine/snapshot_net.js --rollback <name>   # restore (asks y/N)
Contribute self-play from a computer

Multi-core headless trainer — pulls the net, self-plays + trains, pushes it back. Far faster than a phone.

CZ_TOKEN=<github-pat> node engine/az_contribute.js [gamesPerIter=500] [sims=50] [pushEvery=5]

eval_zero.js's hard bot is a vendored snapshot of the cribbage-trainer heuristic (src/winprob.js + src/engine.js's pegChooseDeep) — a fixed benchmark that may drift from the shipped one. Public domain (The Unlicense).
