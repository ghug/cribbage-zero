♣ Cribbage Zero

Browser tools

On-device trainer →local.html — self-plays + trains on this device, pushes the net to GitHub. Cloudflare worker mode (actor) →worker.html — this device self-plays as an actor: reads the net from GitHub, uploads sample shards to the data bus for the learner. Evaluate the net →eval.html — 10000 balanced games vs random + vs hard; recording to the progress branch is a separate manual approval. View progress →progress.html — graphs net win% over games trained from the progress branch. Net snapshots & rollback →snapshot.html — archive the net to the net-archive branch, or roll back to a saved snapshot. Refresh the game's net →refresh-trainer.html — triggers cribbage-trainer's workflow to ship the latest net to its "Zero" bot. Release the game →release-trainer.html — launches a full cribbage-trainer release (signed APK). Confirms first.

Command-line tools

Evaluate & record strength

Plays 10000 balanced games vs random + 10000 vs the hard bot, then asks before appending a point to progress-random.csv / progress-hard.csv on the progress branch (y/N). Recording needs ≥ 10000 games.

CZ_TOKEN=<github-pat> node engine/eval_zero.js [decks=5000]   # prompts before recording
node engine/eval_zero.js --dry          # eval & print, never record
CZ_TOKEN=<pat> node engine/eval_zero.js --yes      # approve non-interactively
Snapshot / roll back the net

Archive the current net to net-archive (a restore point), list snapshots, or roll one back onto the live net branch. The net branch itself keeps no history, so these are your backups.

CZ_TOKEN=<pat> node engine/snapshot_net.js                    # archive now
node engine/snapshot_net.js --list                             # list snapshots
CZ_TOKEN=<pat> node engine/snapshot_net.js --rollback <name>   # restore (asks y/N)
Contribute self-play from a computer (learner / actor)

Multi-core headless self-play. As the learner it trains a bounded replay buffer with shuffled mini-batches, drains the data bus, and pushes the net every ~10k games (+ wind-down). With --actor it only self-plays and uploads sample shards to the bus (reads the net from GitHub, no token). One learner, many actors — see docs/BUS.md.

CZ_TOKEN=<pat> node engine/az_contribute.js [gamesPerRound=500] [sims=40]            # learner (solo)
CZ_BUS_URL=.. CZ_BUS_TOKEN=<trainer> CZ_TOKEN=<pat> node engine/az_contribute.js         # learner + bus
CZ_BUS_URL=.. CZ_BUS_TOKEN=<worker>  node engine/az_contribute.js --actor                # actor
# env: CZ_PUSH_GAMES CZ_BUF CZ_BATCH CZ_WORKERS CZ_SHARD_MAX CZ_BUS_LIMIT

eval_zero.js's hard bot is a vendored snapshot of the cribbage-trainer heuristic (src/winprob.js + src/engine.js's pegChooseDeep). Version · public domain (The Unlicense).
